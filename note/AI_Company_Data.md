# 各AI公司自有数据情况分析

> 本笔记梳理主流AI公司的训练数据来源、自有数据规模及数据战略。
> 最后更新：2026-04-22

---

## 一句话结论

| 公司 | 自有数据 | 规模 | 公开程度 |
|------|----------|------|----------|
| **Google** | ✅ 全球最强 | 最大 | 不公开 |
| **Meta** | ✅ 很强（社交数据） | 极大 | 部分公开 |
| **OpenAI** | ⚠️ 一般（用户交互数据） | 中等 | 不详细公开 |
| **Anthropic** | ⚠️ 一般 | 中等 | 不公开 |
| **DeepSeek** | ❌ 较弱 | 有限 | 公开 |
| **xAI** | ❌ 较弱 | 早期 | 有限 |

---

## 一、Google（最强数据持有者）

### 自有数据来源

Google 拥有全球最大的私有数据池：

| 数据源 | 体量 | 说明 |
|--------|------|------|
| **Google Search** | 数十亿级 | 全球最大搜索引擎数据 |
| **YouTube** | 10亿+ 用户视频 | 全球最大视频平台 |
| **Gmail** | 数十亿邮件 | 全球最大邮箱服务 |
| **Google Books** | 4000万+ 图书 | 全球最大数字图书馆 |
| **Google Scholar** | 数亿学术论文 | 学术引用数据 |
| **Google Maps** | 全球地理数据 | 街景、地图、商户 |
| **Android** | 数十亿设备 | 用户行为数据 |
| **Google Cloud** | 企业数据 | B端数据 |

### Gemini 的训练数据（据官方 Model Card）

> "publicly-available web-documents + licensed data via commercial agreements + user data from Google products + synthetic data"

**Gemini 3 明确承认使用：**
1. 公开可下载的数据集
2. 网络爬取的数据
3. **商业授权数据**（花钱买的数据）
4. **Google 产品用户数据**（你的 Gmail、YouTube 观看记录等）
5. 合成数据（AI 自生成的数据）

### Google 的数据优势总结

- **规模**：全球无敌，没有任何公司能匹敌
- **多样性**：文本、代码、视频、音频、图书、地理位置全覆盖
- **质量**：学术论文、书籍等高质量内容占比高
- **壁垒**：搜索引擎+云服务+社交+硬件形成数据护城河

## 二、Meta

### Llama 1-2 时期：宣称"纯公开数据"

Meta 早期强调只用公开数据：

> "It is possible to train state-of-the-art models using publicly available datasets exclusively" — Llama 论文

**数据来源**（Llama 2）：

| 来源 | 比例 | 说明 |
|------|------|------|
| Common Crawl | 67% | 网页文本 |
| C4 | 15% | 清洗后网页 |
| GitHub | 4.5% | 代码 |
| Wikipedia | 4.5% | 百科 |
| Books | 4.5% | 图书 |
| ArXiv | 2.5% | 学术论文 |
| StackExchange | 2% | 技术问答 |

### Llama 3-4 转变：开始使用自有产品数据

Llama 4 官方披露：

> "Llama 4 Scout 预训练于 ~40T tokens，Llama 4 Maverick 预训练于 ~22T tokens——来自公开数据、授权数据和 **Meta 产品信息** 的混合"

**Meta 自有数据来源：**
- **Facebook** 用户帖子+互动数据
- **Instagram** 图片+描述+评论
- **Meta AI** 用户交互数据

### 为什么 Meta 态度转变

1. 公开高质量数据被竞争对手用光了
2. 自有社交数据价值巨大且不依赖外部
3. 授权数据越来越贵
4. 合成数据成为新方向

## 三、OpenAI

### GPT-3 时期（相对透明）

GPT-3 训练数据来源是公开的：

| 数据集 | tokens | 比例 |
|--------|--------|------|
| Common Crawl（过滤版） | 4100亿 | 60% |
| WebText2 | 190亿 | 22% |
| Books1 | 120亿 | 8% |
| Books2 | 550亿 | 8% |
| Wikipedia | 30亿 | 3% |

### GPT-4 之后：不详细公开

GPT-4 官方说法：

> "publicly available data (such as internet data) and data licensed from third-party providers"

- 公开数据（Common Crawl 等）
- **第三方授权数据**（花钱买的数据）
- 不再详细披露各来源比例

### OpenAI 的用户数据使用

根据 2026年3月更新的官方政策：

| 用户类型 | 数据用于训练？ |
|----------|--------------|
| ChatGPT 个人用户 | ✅ 默认可用于训练，可关闭 |
| ChatGPT Team/Enterprise | ❌ 默认不训练 |
| API 用户 | ❌ 默认不训练，可主动 opt-in |

### OpenAI 的数据特点

- **优势**：通过 ChatGPT 用户交互积累了大量真实对话数据
- **劣势**：没有 Google 那样的搜索引擎护城河
- **策略**：通过 API 和产品收集用户交互数据，同时购买授权数据

## 四、Anthropic

### Claude 的数据

Anthropic 对训练数据披露极少，只提到：
- 公开可用的数据
- 授权数据
- 用户交互数据（可通过设置关闭）

**Claude 3 Model Card 说法：**

> "Anthropic may use public and private datasets for training, including data from the Claude API and Claude.ai. Users can request to exclude their data from training."

### Anthropic 的数据战略

1. **宪法AI（Constitutional AI）** 方法减少对大规模人类标注数据的依赖
2. 通过 RLHF 和人类反馈降低数据需求
3. 强调安全性和对齐，数据质量优先于数量
4. 融资超 60 亿美元，有能力购买高质量授权数据

---

## 五、DeepSeek

### DeepSeek 的数据情况

DeepSeek 选择了一条不同的路——**主要依赖公开数据**：

- 训练数据以公开可用的数据集为主
- 使用 Common Crawl、Wikipedia、书籍、代码等
- 强调算法优化而非数据规模
- V3 的训练仅用 2048 个 H800 GPU，效率极高

### DeepSeek 的优势与局限

| | 说明 |
|---|---|
| **优势** | 成本低、效率高、数据合规风险低 |
| **劣势** | 没有自有数据护城河，容易被复制 |

**简单说**：DeepSeek 是"算法换数据"的路线，数据方面反而是弱项。

## 六、百度（文心一言）

### 自有数据来源

| 数据源 | 说明 |
|--------|------|
| **百度搜索** | 中文互联网最大搜索引擎数据 |
| **百度贴吧** | 大量中文社区讨论 |
| **百度知道/百科** | 问答+百科全书 |
| **爱奇艺** | 视频内容（百度旗下） |
| **Apollo** | 自动驾驶数据 |

### 训练数据披露情况

根据公开资料，文心一言的训练数据：

- 主要是**互联网公开数据**（百度官方曾表示"使用互联网公开数据，符合行业惯例"）
- 中文语料占比高，针对中文场景做了专项优化
- 文心一言4.5论文显示：预训练数据2.3TB，中文占比89%

### 百度的数据特点

| 特点 | 说明 |
|------|------|
| **中文优势** | 百度搜索+贴吧+知道，覆盖大量中文高质量语料 |
| **搜索数据** | 用户搜索行为可反映真实信息需求 |
| **弱项** | 没有全球社交数据，英文数据依赖公开语料 |

## 七、腾讯（混元大模型）

### 自有数据来源

| 数据源 | 说明 |
|--------|------|
| **微信** | 全球最大即时通讯，聊天数据、朋友圈 |
| **QQ** | 社交通话数据 |
| **腾讯游戏** | 游戏内交互数据 |
| **腾讯视频** | 视频内容（长视频） |
| **QQ浏览器** | 搜索+内容消费数据 |
| **企业微信** | B端办公数据 |

### 训练数据披露情况

根据官方资料：

- 混元大模型预训练语料超 **2万亿 tokens**
- 主要来源：公开数据 + 合作伙伴数据 + **腾讯自身产品数据**
- 官方承认使用腾讯内部产品数据训练
- 腾讯广告推荐系统积累了大量用户行为数据

### 腾讯的数据特点

| 特点 | 说明 |
|------|------|
| **社交数据** | 微信+QQ 是全球最大即时通讯数据之一 |
| **用户行为** | 游戏、社交、广告点击数据 |
| **弱项** | 搜索引擎数据不如百度，英文数据有限 |

## 八、月之暗面（Kimi / Moonshot）

### 自有数据来源

Kimi 是一家成立于2023年3月的创业公司，创始人杨植林曾任职 Google 和 Meta AI。

| 数据源 | 说明 |
|--------|------|
| **公开语料** | 主要依赖公开可用的互联网数据 |
| **长上下文数据** | Kimi 强项，支持100万token上下文 |
| **阿里云** | Kimi 使用阿里云进行数据预处理 |

### Kimi K2 的训练数据

根据官方技术报告：

- K2 预训练于 **15.5T tokens**
- 数据来源：网页文本、代码、数学数据
- 使用了合成数据（Synthetic Data）管道进行 AI Agent 训练
- 数据重述（Data Rephrasing）技术提升质量

### Kimi 的数据特点

| 特点 | 说明 |
|------|------|
| **公开数据为主** | 没有自有产品数据护城河 |
| **合成数据** | 通过自研合成数据管道扩展训练数据 |
| **长上下文** | 对长文本数据有专项优化 |

## 十、MiniMax（海螺AI）

### 自有数据来源

MiniMax 是中国估值最高的大模型创业公司之一，2025年已完成巨额融资并传出港股上市。

| 数据源 | 说明 |
|--------|------|
| **语音数据** | MiniMax 是极少数押注语音大模型的团队 |
| **长音频训练** | 数百万小时高质量音频数据训练语音模型 |
| **公开语料** | 主要依赖公开可用数据 |
| **阿里云** | 使用阿里云进行数据预处理 |

### MiniMax 的训练数据

根据公开资料：

- abab 6.5：万亿参数 MoE，200K 上下文
- 海螺AI 产品背后接入自研多模态大模型
- 语音大模型效果不输 ElevenLabs 和 OpenAI
- M2.7 / M1 等新模型持续迭代

### MiniMax 的数据特点

| 特点 | 说明 |
|------|------|
| **语音数据** | 核心差异化，数据量达"数百万小时" |
| **产品驱动** | 海螺AI 用户交互数据积累 |
| **弱项** | 没有全球社交数据 |

## 七、为什么大公司不公开训练数据

### 1. 竞争壁垒
数据是核心护城河。Google 之所以能训练 Gemini，搜索引擎和 YouTube 的数据是独家优势。公开等于送给竞争对手。

### 2. 法律风险
版权争议。各AI公司正面临大量版权诉讼，公开数据会让法律风险更清晰。

### 3. 商业利益
数据可以单独授权变现。比如 Google 向企业卖 Google Cloud 数据服务。

### 4. 用户隐私
Google 的用户数据即使匿名化，公开也会引发隐私争议。

## 八、真正公开训练数据的只有这几类

| 类型 | 代表 | 公开原因 |
|------|------|----------|
| **非营利机构** | AI2（Dolma） | 使命驱动，不靠数据盈利 |
| **学术机构** | BigScience（BLOOM）、Mila | 研究驱动 |
| **国家项目** | Apertus（瑞士） | 主权AI战略，需合规透明 |
| **开源社区** | EleutherAI（The Pile） | 打破闭源垄断 |

---

## 九、总结

**Google** 拥有全球最强的私有数据池（搜索+视频+邮箱+图书），是AI训练数据的隐形冠军。

**Meta** 拥有全球最大的社交数据（Facebook+Instagram），Llama 4 开始承认利用这些数据。

**OpenAI** 靠用户交互数据+授权数据，没有独占数据护城河。

**Anthropic** 强调安全和质量，数据策略保守。

**DeepSeek** 主要靠公开数据，数据方面反而是短板。
