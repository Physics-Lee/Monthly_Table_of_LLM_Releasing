# LLM 本地/半开源工具简介

> 本文档记录近期加入 LLM-Applications 和 Open-Source 列的四个工具
> 更新时间: 2026-04-20

---

## GPT4All

| 项目 | 信息 |
|------|------|
| **定位** | 本地运行大语言模型的桌面应用 |
| **发布时间** | 2023-03-27（GitHub 创建） |
| **开源协议** | MIT |
| **开发方** | Nomic AI |
| **GitHub** | https://github.com/nomic-ai/gpt4all |
| **官网** | https://docs.gpt4all.io/ |

**功能简介：**
- 让消费者级电脑（无 GPU 也可）本地运行 LLM
- 内置模型下载器，支持 Llama、Mistral、Gemma 等主流开源模型
- 支持本地文档（LocalDocs）—— 上传文件后让模型基于文件内容回答
- 有 Python SDK，可编程调用
- 数据完全本地化，无需联网

**为什么是"半开源"：**
- 代码完全开源（MIT），可以审计、Fork、二次开发
- 但本身不是模型，而是模型**运行平台**——依赖第三方开源模型
- 类似浏览器之于网页的关系

## LM Studio

| 项目 | 信息 |
|------|------|
| **定位** | 本地 LLM 桌面客户端 + API 服务器 |
| **发布时间** | 2023 年早期版本，2024-05 广泛发布 |
| **开源协议** | **闭源**，但个人和商业使用免费 |
| **开发方** | Element Labs |
| **官网** | https://lmstudio.ai/ |

**功能简介：**
- 比 GPT4All 更成熟的 GUI，支持聊天、模型管理、参数调节
- 内置 llama.cpp / MLX 推理引擎
- 可开启本地 OpenAI-compatible API 服务器（让其他应用调用本地模型）
- 支持多模态（图片理解）、工具调用（Tool Use）、结构化输出
- 2026-01 发布 0.4.0，推出无 GUI 的 headless 版本 `llmster`

**为什么是"半开源"：**
- **软件本身闭源**——无法查看或修改 LM Studio 的代码
- 但**免费使用**（2025-07 起商业用途也免费）
- 依赖的开源模型（Llama、Qwen 等）是完全开源的
- 属于"开源生态的闭源前端"

## AnythingLLM

| 项目 | 信息 |
|------|------|
| **定位** | 全栈 AI 生产力平台（本地/远程均可） |
| **发布时间** | 2023-06-04（GitHub 创建） |
| **开源协议** | MIT |
| **开发方** | Mintplex Labs |
| **GitHub** | https://github.com/Mintplex-Labs/anything-llm |
| **官网** | https://anythingllm.com |

**功能简介：**
- 支持几乎所有 LLM：OpenAI、Anthropic、本地 Ollama/LM Studio、云 API 等
- **RAG（检索增强生成）**：上传文档后让模型基于文档回答
- **AI Agent**：可创建自定义 Agent，支持工具调用、网络搜索
- **MCP 支持**：兼容 Model Context Protocol
- 有桌面版（Mac/Windows/Linux）和 Docker 服务器版
- 内置向量数据库（LanceDB），无需额外配置

**为什么是"半开源"：**
- 代码完全开源（MIT）
- 但可以连接**闭源商业模型**（GPT-4、Claude 等）
- 本身是一个**平台/框架**，不训练自己的模型
- 属于 LLM 应用层基础设施

## RAGFlow

| 项目 | 信息 |
|------|------|
| **定位** | 开源 RAG 引擎 + Agent 框架 |
| **发布时间** | 2023-12-12（GitHub 创建） |
| **开源协议** | Apache-2.0 |
| **开发方** | InfiniFlow |
| **GitHub** | https://github.com/infiniflow/ragflow |
| **Demo** | https://cloud.ragflow.io |

**功能简介：**
- **深度文档解析**：对 PDF、DOCX 等复杂格式做精细化解析，不是简单切片
- **可编排的 RAG Pipeline**：支持多阶段检索、重排序、引用溯源
- **Agent 工作流**：支持 Agent + MCP + 代码执行器（Python/JS）
- **多模态 RAG**：支持图片理解、表格提取
- 自研文档引擎 Infinity（AI-native 数据库）

**为什么是"开源"（而非半开源）：**
- 代码完全开源（Apache-2.0）
- 自研核心组件（Infinity 向量数据库也是开源的）
- 但同样**不训练自己的 LLM**，依赖外部模型
- 属于 LLM 生态的**中间件层**

## 一句话区分

| 工具 | 核心卖点 | 适合谁 |
|------|----------|--------|
| **GPT4All** | 最简单——下载即用的本地聊天工具 | 个人用户，想快速体验本地 LLM |
| **LM Studio** | 最成熟——功能最全的本地模型管理器 | 开发者，需要 API 服务或高级功能 |
| **AnythingLLM** | 最灵活——支持几乎所有模型和部署方式 | 团队/企业，需要 RAG + Agent |
| **RAGFlow** | 最深入——文档解析和 RAG Pipeline 最强 | 需要高质量文档问答的场景 |

## 开源程度对比

| 工具 | 代码开源 | 核心功能开源 | 自研模型 | 商业化 |
|------|---------|-------------|----------|--------|
| GPT4All | ✅ MIT | ✅ | ❌ | 免费 |
| LM Studio | ❌ | ❌ | ❌ | 免费（闭源） |
| AnythingLLM | ✅ MIT | ✅ | ❌ | 免费 + 付费托管版 |
| RAGFlow | ✅ Apache-2.0 | ✅ | ❌ | 免费 + 云托管版 |

> 四个工具**都不训练自己的 LLM**，它们都是 LLM 生态的**应用层/中间件层**。
> 这种"用开源模型 + 自己做平台"的模式是当前 AI 应用的主流路径。
