## 1. 项目定位

**LLM Release Timeline** —— 一个追踪全球大模型发布动态的静态信息聚合网站。

- **在线地址**：https://physics-lee.github.io/Monthly_Table_of_LLM_Releasing/
- **GitHub 仓库**：https://github.com/Physics-Lee/Monthly_Table_of_LLM_Releasing
- **时间跨度**：2022-11 ~ 2026-04（持续更新）
- **核心形态**：一张超宽的横向时间线表格，按月份 × 厂商二维排列

---

## 2. 技术架构

| 层级 | 技术 | 说明 |
|---|---|---|
| 前端 | 原生 HTML + CSS + JS | 无框架，单页应用 |
| 数据 | `data.json` + `links.json` | JSON 静态文件 |
| 构建 | Node.js 脚本 | 从 CSV/MD 生成 JSON |
| 部署 | GitHub Pages | 推送到 main 分支即自动部署 |

**无后端、无数据库、无构建工具链**（如 Webpack/Vite）。极简静态站点。

---

## 3. 核心文件职责

| 文件 | 职责 |
|---|---|
| `index.html` | 页面骨架，引用 CSS/JS |
| `styles.css` | 完整样式，含响应式设计、暗色变量预留 |
| `app.js` | **核心逻辑**：加载数据 → 初始化筛选器 → 渲染表格 → 绑定事件 |
| `data.json` | **核心数据**：`{ vendors: [], rows: [] }`，35个厂商 × N个月 |
| `links.json` | 模型名称 → 官方发布链接的映射，用于表格中超链接跳转 |
| `check-links.js` | 独立工具：检查 links.json 中是否有失效链接 |
| `fix-baidu-iflytek-links.js` | 针对百度和讯飞链接的特殊修复脚本 |
| `llm_release_timeline_*.csv` | 数据源的 CSV 格式 |
| `llm_release_timeline_*.md` | 数据源的 Markdown 格式 |
| `README.md` | 项目主页说明，内含完整的静态表格（GitHub 直接渲染） |

---

## 4. 前端交互能力（app.js）

1. **搜索过滤**：输入模型名称，实时过滤行
2. **年份筛选**：下拉选择 2022~2026，或全部
3. **厂商多选**：35个厂商的 checkbox 组，可开关显示列
4. **时间排序**：正序/倒序切换按钮
5. **链接渲染**：单元格内模型名自动匹配 `links.json` 生成超链接
6. **更新日志**：页面底部自动拉取 GitHub API 最近 3 条 commit 记录
7. **厂商中文映射**：`VENDOR_NAMES` 对象，英文 → 中文别名

## 6. 数据维护工作流

```
CSV/MD 源文件（人工编辑）
    ↓
scripts/build-json.js → 生成 data.json
    ↓
scripts/generate-readme.js → 生成 README.md 中的表格
    ↓
scripts/check-missing-links.js → 检查是否有新模型缺链接
    ↓
scripts/fix-md-links.js / update-md-links.js → 修复/更新链接
    ↓
git commit + push → GitHub Pages 自动部署
```

## 8. 关键观察

1. **数据权威性**：依赖各厂商官方发布渠道及公开报道，中国厂商数据部分来自官方里程碑页面或开发者发布说明。
2. **开源协作**：README 声明"由社区维护，欢迎提交 Issue 和 Pull Request"。
3. **表格密度**：42个月 × 35列 = 近 1500 个单元格，信息密度极高。
4. **近期活跃**：最新数据已覆盖到 2026-04（GPT-5.4-Cyber、Claude Opus 4.7、Qwen3.6 等），维护者更新非常及时。
5. **无持久化状态**：前端所有筛选状态（搜索词、选中厂商、年份、排序）均为页面级，刷新即重置，无 URL query 参数持久化。
