## 一、重要前提：你的项目是否适合直接上传 Coze

读完官方文档后，我发现一个**关键信息**：

> 官方文档讲的 "扣子编程" 主要是 **AI 自动生成代码** 的模式——你跟 AI 对话，它帮你写网页应用。

但 **`wmpeng/codingplan` 的模式不一样**——它是**直接上传已有的静态网站文件**（HTML/CSS/JS + `.coze` 配置文件）。

这两种方式在 Coze 上的入口可能不同：

| 方式 | 说明 | 适合场景 |
|------|------|---------|
| **扣子编程（Vibe Coding）** | 跟 AI 对话，让它从零生成网页应用 | 没有现成代码，想让 AI 帮你写 |
| **上传现有项目（Coding 模式）** | 直接上传本地代码文件夹，用 `.coze` 配置运行 | 已经有写好的代码，像 `codingplan` 那样 |

**你的情况**：已经有写好的 `index.html` + `styles.css` + `app.js` + `.coze`，应该选择 **上传现有项目** 的方式。

## 二、部署步骤（针对已有代码的项目）

根据 `codingplan` 的模式 + Coze 部署文档，操作步骤如下：

### Step 1：进入扣子编程 / 项目管理

1. 打开 [coze.cn](https://www.coze.cn) 并登录
2. 在左侧导航栏选择 **"项目管理"**
3. 点击 **"创建项目"** 或 **"新建网页应用"**

### Step 2：选择正确的创建方式

这里有两种可能的路径：

#### 路径 A：直接有"上传项目"入口（推荐）
- 在创建页面寻找 **"导入项目"**、**"上传代码"** 或 **"从本地导入"** 的选项
- 选择你的项目文件夹（包含 `index.html` 和 `.coze` 的根目录）

#### 路径 B：先创建空白项目，再替换代码
- 如果找不到直接上传入口：
  1. 先随便创建一个空白网页应用
  2. 进入 **AI 编程开发界面**
  3. 在左侧文件树中，找到上传/导入文件的按钮
  4. 把 `index.html`、`styles.css`、`app.js`、`data.json`、`links.json`、`.coze` 全部上传
  5. 删除自动生成的多余文件

### Step 3：确认入口文件和运行配置

- **入口文件**：确保是 `index.html`
- **运行配置**：Coze 应该自动读取 `.coze` 文件中的配置：
  ```toml
  [project]
  entrypoint = "index.html"
  requires = ["python-3.12"]

  [deploy]
  build = []
  run = ["python", "-m", "http.server", "5000", "--bind", "0.0.0.0"]
  ```

### Step 4：预览测试

- 在开发界面中点击 **"预览"** 或 **"运行"**
- 检查页面是否正常加载：
  - 表格是否正确渲染
  - 筛选器是否正常工作
  - 中文是否显示正常
  - 链接是否可以点击

### Step 5：部署发布

1. 点击右上角的 **"部署"** 按钮
2. **配置域名**：
   - 使用默认的 `xxx.coze.site` 域名（免费）
   - 或添加自定义域名（需要备案，较麻烦）
3. **配置可见性**：
   - 选择 **"公开部署"**（当前仅支持公开访问）
4. **数据库同步**：
   - 你的项目是纯静态网站，**没有数据库**，这一步可以忽略或选择关闭
5. **环境变量**：
   - 纯静态网站不需要配置
6. 点击 **"开始部署"**

### Step 6：获取访问链接

- 部署成功后，复制生成的 URL
- 把这个 URL 填回 `scripts/generate-readme.js` 中的 `ONLINE_URL`
- 重新运行 `node scripts/generate-readme.js` 更新 README
- 推送更新到 GitHub

## 三、费用与配额

### 费用
- **部署和访问**：**免费**
- 纯静态网页应用**不消耗**扣子积分

### 配额限制
- 可创建的项目数量有限制
- 可部署的次数有限制
- 具体额度查看官方「配额与限制」页面

**对于你的场景**：一个月更新 1-2 次，免费额度完全够用。

## 四、常见问题

### Q1：找不到"上传本地项目"的入口怎么办？

Coze 的界面可能随版本调整。如果左侧没有明显的上传按钮：
- 尝试在文件树区域右键 → "上传文件"
- 或寻找一个 "导入" / "Import" 图标
- 如果都没有，可能需要先创建空白项目，再在代码编辑器里逐个文件新建并粘贴内容

### Q2：中文显示乱码怎么办？

- 确认 `index.html` 的 `<head>` 中有：
  ```html
  <meta charset="UTF-8">
  ```
- 我们的代码已经包含这个标签，浏览器会自动正确识别

### Q3：部署后链接打不开？

- 检查 `data.json` 和 `links.json` 是否也一起上传了
- 检查浏览器控制台（F12）是否有 404 错误
- 确认文件路径是相对路径（如 `./data.json` 而不是绝对路径）

### Q4：如何更新已部署的网站？

1. 进入 Coze 的项目管理页面
2. 找到你的项目，进入编辑
3. 修改文件或重新上传
4. 再次点击"部署"
5. 覆盖之前的版本

## 六、总结

你的项目可以直接上传到 Coze 部署，核心步骤是：

1. **登录 Coze** → 项目管理 → 创建/上传项目
2. **上传所有静态文件**（HTML/CSS/JS/JSON/.coze）
3. **预览测试** → 确认功能正常
4. **点击部署** → 选择公开访问
5. **复制链接** → 更新 README 和 GitHub
