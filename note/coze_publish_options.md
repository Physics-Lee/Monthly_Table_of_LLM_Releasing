# 用扣子（Coze）发布 LLM Release Timeline 的两种方案

> 分析时间：2026-04-14  
> 适用项目：`llm_release_timeline`（大模型发布时间线）

---

## 核心结论

**费用：¥0（完全免费）**

纯静态信息展示页（Markdown 表格、链接、无 AI 对话、无模型调用）不消耗 Coze 积分，因此无需付费。

---

## 方案 A：卡片式网页（极简版）

### 适合人群
- 不想写任何代码
- 5 分钟内快速上线
- 对页面美观度要求不高

### 操作步骤

1. **注册/登录** [coze.cn](https://www.coze.cn)
2. **创建应用**：选择「卡片」或「网页」类型
3. **粘贴内容**：把 `llm_release_timeline_2022-11_to_2026-04.md` 的 Markdown 表格直接贴进内容区
4. **设置公开**：发布为「公开可访问」
5. **获得链接**：形如 `https://xxxxx.coze.site`

### 优点
- 零代码，上手极快
- 国内访问稳定
- 完全免费

### 缺点
- 页面格式简陋，不像专业数据网站
- 表格可能显示不全或需要横向滚动
- 大概率带 Coze 品牌水印（如底部 "Powered by Coze"）
- 自定义程度极低
- 数据更新后需手动回到 Coze 后台重新粘贴

---

## 方案 B：完整静态网站部署（推荐版）

参考 `wmpeng/codingplan` 的做法，把自建的 HTML/CSS/JS 静态网站打包上传到 Coze，用 Python HTTP 服务器运行。

### 适合人群
- 希望页面专业、可交互
- 愿意花 30 分钟搭建
- 未来可能持续迭代功能（筛选、搜索、可视化等）

### 项目结构

```
llm_release_timeline/
├── .coze                    # Coze 部署配置文件
├── index.html               # 主页面（交互式时间线/表格）
├── data.json                # 从 CSV 转换来的结构化数据
├── styles.css               # 自定义样式
├── app.js                   # 页面交互逻辑（筛选、渲染等）
└── README.md                # 项目说明（可选）
```

### `.coze` 配置文件示例

```ini
[project]
entrypoint = "index.html"
requires = ["python-3.12"]

[dev]
build = []
run = ["python", "-m", "http.server", "5000", "--bind", "0.0.0.0"]

[deploy]
build = []
run = ["python", "-m", "http.server", "5000", "--bind", "0.0.0.0"]
```

### 操作步骤

1. **准备静态网站文件**
   - 将 CSV 数据转换为 `data.json`
   - 编写 `index.html`，用 JavaScript 读取 `data.json` 并动态渲染表格
   - 编写 `styles.css` 美化页面
   - 编写 `app.js` 实现筛选、高亮、展开详情等交互
   - 根目录放置 `.coze` 配置文件

2. **上传到 Coze**
   - 在 Coze 创建「网页应用」或「自定义项目」
   - 上传整个项目文件夹
   - 确认入口文件为 `index.html`

3. **发布**
   - 点击部署/发布
   - 获得公开访问链接

### 页面可具备的功能

| 功能 | 说明 |
|------|------|
| **按厂商筛选** | 勾选框控制显示/隐藏特定厂商列 |
| **按年份筛选** | 只显示 2024、2025 等特定年份 |
| **模型高亮** | 鼠标悬停显示该模型的详细信息 |
| **来源链接** | 点击模型名跳转官方发布页面 |
| **响应式布局** | 适配手机和桌面端 |
| **搜索框** | 快速搜索某个模型名称 |

### 优点
- 页面效果专业，完全可控
- 支持丰富的交互功能
- 数据更新只需替换 `data.json`，无需改页面代码
- 国内访问稳定，且免费
- 可扩展性强（未来可加图表、API、多语言等）

### 缺点
- 需要写一些前端代码（HTML/CSS/JS）
- 首次搭建比方案 A 多花 30 分钟到 1 小时

---

## 两种方案对比

| 维度 | 方案 A：卡片式网页 | 方案 B：完整静态网站 |
|------|------------------|-------------------|
| **费用** | ¥0 | ¥0 |
| **技术门槛** | 无 | 低（需基础 HTML/CSS/JS） |
| **搭建时间** | 5 分钟 | 30-60 分钟 |
| **页面美观度** | 一般 | 高 |
| **交互功能** | 无 | 可自定义筛选、搜索、高亮 |
| **品牌水印** | 可能有 Coze 水印 | 无 |
| **自定义域名** | 不支持 | 暂不支持（可用 Coze 提供的链接） |
| **数据更新方式** | 手动粘贴 | 替换 JSON 文件 |
| **长期可维护性** | 差 | 好 |
| **推荐度** | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ |

---

## 最终建议

- **如果只是临时分享一次** → 选 **方案 A**，粘贴即走。
- **如果想长期维护、看起来专业、未来还能加功能** → 选 **方案 B**，模仿 `codingplan` 的模式。

如果决定做方案 B，下一步可以：
1. 把 `llm_release_timeline_2022-11_to_2026-04.csv` 转成 `data.json`
2. 设计一个简洁的交互式时间线页面
3. 配置 `.coze` 文件并上传到 Coze 发布
