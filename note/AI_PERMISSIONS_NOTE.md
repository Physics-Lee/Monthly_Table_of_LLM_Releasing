# 关于 AI 代理的系统权限说明

## 权限来源

我（AI 代理）**没有特殊的系统权限**。我执行命令时使用的是：

- **你的用户账户权限**
- **当前登录会话的权限**
- **和你手动打开终端完全相同的权限**

## 为什么我能执行 git 命令？

```
你（用户） → 启动 AI 代理 → 调用 bash 工具 → 系统 shell → 执行命令
```

这个调用链等价于你自己在终端输入命令。

## 具体说明

| 操作 | 权限来源 | 等价操作 |
|---|---|---|
| `git push` | 你的 SSH key / HTTPS 凭证 | 你自己运行 `git push` |
| 文件写入 | 你的用户文件系统权限 | 你自己保存文件 |
| 网络请求 | 你的网络环境 | 你自己浏览器访问 |

## 安全风险提醒

⚠️ **我能做的事情 = 你能做的事情**

- 我能访问你当前用户能访问的所有文件
- 我能执行你当前用户能执行的所有命令
- 我能修改你当前用户能修改的所有内容

## 建议

1. **敏感操作前确认**：涉及代码提交、文件删除、密钥操作等，应明确授权
2. **使用版本控制**：所有修改都在 git 跟踪下，可回滚
3. **最小权限原则**：重要仓库可设置保护分支（require PR review）
4. **定期审查**：查看 `git log` 确认所有提交都是你预期的

## 本次会话中的 git 操作

所有提交都使用你的本地 git 配置（用户名/邮箱），推送到你授权的远程仓库。

如需限制权限，可：
- 禁用 bash 工具（只使用文件读写）
- 设置 git  hooks 进行提交前审查
- 使用只读环境运行 AI 代理
