# 将 LLM Release Timeline 发布到公开网站的方案对比

> 需求：把 `llm_release_timeline_2022-11_to_2026-03_v3.csv` 和 `llm_release_timeline_2022-11_to_2026-03_v3.md` 发布成一个公开网站，让所有人（尤其是国内用户）都能在互联网上访问。
>
> 限制：不能使用 GitHub Pages（已用于个人网站，且国内访问不稳定）。

---

## 方案一：Coze（扣子）发布网页

### 费用
**¥0（免费）**

### 为什么免费
Coze 的收费点主要是：
- 模型调用（Bot 对话消耗积分）
- API 调用（把 Bot 当接口用）
- 工作流测试/运行（涉及 LLM 节点时）

你的 LLM Release Timeline 是**纯静态信息展示页**：
- 一个 Markdown 表格
- 一堆来源链接
- 没有 AI 对话功能
- 不需要实时调用大模型

如果做成 Coze 的**卡片式应用/网页发布**，属于静态展示，**不消耗模型积分，因此不收费**。

### 两种方式

| 方式 | 说明 | 费用 | 缺点 |
|------|------|------|------|
| **卡片式网页** | 直接把 Markdown 内容贴进 Coze 应用，发布为网页 | ¥0 | 页面格式简陋，不像专业数据表格 |
| **工作流动态生成** | 用户访问时让 Bot 读取 CSV 并生成回答 | 按调用次数扣积分 | 静态数据没必要动态生成，访问量大了会花钱 |

### 主要缺点
- **不是专业静态网站托管平台**，页面功能受限
- 可能带有 Coze 品牌水印（如底部"Powered by Coze"）
- 域名是 `xxx.coze.site`，不够独立
- 数据更新后需要手动回到 Coze 平台修改内容

---

## 方案二：Gitee Pages（强烈推荐）

### 费用
**¥0（完全免费）**

### 优点
- 和 GitHub Pages 机制几乎一样，但**服务器在国内**
- **国内访问速度极快**
- 支持自定义域名
- 自动渲染 Markdown
- 推送代码后自动部署

### 操作流程
1. 注册/登录 [gitee.com](https://gitee.com)
2. 新建仓库，把两个文件推送上去
3. 开启 **Gitee Pages** 服务
4. 获得 `https://你的用户名.gitee.io/仓库名` 的公开网址
5. 把 `llm_release_timeline_2022-11_to_2026-03_v3.md` 重命名为 `README.md` 可直接渲染

### 适合场景
已经有 Git 仓库、想要一个**专业、稳定、国内可访问**的静态网站。

---

## 方案三：Coding Pages

### 费用
**¥0（完全免费）**

### 优点
- 腾讯旗下平台，**国内访问稳定**
- 支持持续集成/自动部署
- 可以绑定自定义域名

### 缺点
- 配置比 Gitee Pages 稍微复杂一点
- 免费版有一些功能限制

---

## 方案四：飞书文档 / 语雀 / Notion 公开页

### 费用
**¥0（完全免费）**

### 优点
- **零技术门槛**，把 Markdown 表格粘贴进去即可
- 一键生成公开分享链接
- 国内访问速度快
- 语雀对技术文档支持很好

### 缺点
- 页面带有平台品牌标识
- 自定义程度低
- CSV 文件需要另外提供下载链接

---

## 方案五：Vercel / Cloudflare Pages

### 费用
**¥0（有免费额度）**

### 优点
- 全球 CDN，技术圈口碑好
- 自动部署，支持自定义域名

### 缺点
- **国内访问不稳定**，偶尔被墙或减速
- 如果主要用户在国内，体验不如 Gitee/Coding

---

## 综合推荐

| 排名 | 方案 | 推荐理由 |
|------|------|---------|
| 🥇 **第一推荐** | **Gitee Pages** | 免费、国内访问快、和 GitHub Pages 一样简单、支持自定义域名 |
| 🥈 第二推荐 | **语雀公开页 / 飞书文档** | 完全零代码，粘贴即用，适合不想折腾的用户 |
| 🥉 第三推荐 | **Coze 卡片式网页** | 确实免费且国内可访问，但页面效果和功能受限，不够专业 |
| ⚠️ 不推荐 | Vercel / Cloudflare Pages | 国内访问不稳定，不符合你的核心需求 |

---

## 结论

> 如果目标是**让国内用户稳定访问、不花钱、且看起来专业**，**Gitee Pages 是最优解**。Coze 虽然免费，但它更适合做 AI Bot 应用，而不是纯静态数据展示网站。
