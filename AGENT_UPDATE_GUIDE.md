## 推荐方式：编辑 Markdown（不易出错）

### 1. 编辑哪个文件

```
llm_release_timeline_2022-11_to_2026-04.md
```

### 2. 格式示例

```markdown
| Month | Open-Source | OpenAI | Anthropic | Google | ... |
|-------|-------------|--------|-----------|--------|-----|
| 25-Apr | OpenCode | GPT-4.1 + GPT-4.1 mini | | | ... |
| 25-May | | GPT-5 | Claude 4 | | ... |
```

**规则：**
- 模型名用 `[模型名](URL)` 格式添加链接
- 多个模型用 ` + ` 分隔（注意空格）
- **关键**：每行的 `|` 分隔符数量必须与表头一致
- 空单元格留空即可（`||`）

### 3. 新增公司/列（重要）

如果要添加新厂商列：

1. **在表头行添加新列名：**
   ```markdown
   | Month | Open-Source | OpenAI | ... | 新厂商 |
   |-------|-------------|--------|-----|--------|
   ```

2. **在所有数据行添加对应列（可以留空）：**
   ```markdown
   | 25-Apr | ... | | 新模型 |
   | 25-May | ... | | |
   ```

3. **保持每行的 `|` 数量一致**（表头有几个 `|`，数据行也要有几个）

### 4. 新增月份/行

如果要添加新月份：

1. 在表格末尾添加新行
2. **复制上一行的 `|` 结构，只修改内容**
3. 第一列是月份（如 `25-Jun`）

### 5. 构建命令

```bash
node scripts/build-json.js
```

**构建成功输出（行列数会变）：**
```
✓ CSV 解析完成: 43 行, 36 列
✓ MD 解析完成: 43 表格行, 315 个链接
✓ 已写入: data.json
✓ 已写入: links.json
✓ 已写入: README.md
✅ 完成！
```

**构建失败输出（必须修复）：**
```
❌ 第 31 行字段数不匹配: 期望 36 列, 实际 35 列
   内容: 25-Apr,OpenCode,GPT-4.1 + GPT-4.1 mini,Gemini 2.5 Flash,...
Error: CSV 格式错误: 第 31 行字段数不匹配
```

## 备选方式：编辑 CSV（需谨慎）

### 1. 编辑哪个文件

```
llm_release_timeline_2022-11_to_2026-04.csv
```

### 2. 核心规则（唯一原则）

```
⚠️ 所有行的逗号数量必须与表头一致
```

- 数一下表头有多少个逗号（当前是 N 个）
- 确保每一行都有 N 个逗号
- 如果单元格内容包含逗号，用双引号包裹：`"a,b,c"`

### 3. 正误示例

```csv
# 表头（假设有 35 个逗号 = 36 列）
Month,Open-Source,OpenAI,...,新厂商

# ✅ 正确：35 个逗号
25-Apr,,GPT-4.1 + GPT-4.1 mini,,,...,

# ❌ 错误：只有 34 个逗号（少了一个）
25-Apr,,GPT-4.1 + GPT-4.1 mini,,,...

# ❌ 错误：有 36 个逗号（多了一个）
25-Apr,,GPT-4.1 + GPT-4.1 mini,,,...,,,
```

### 4. 新增公司/列

1. 在表头添加新列名（加一个逗号 + 列名）
2. **在所有数据行末尾添加一个逗号**（对应新列，即使留空）
3. 重新运行构建验证

### 5. 构建和验证

```bash
node scripts/build-json.js
```

**必须构建成功后再提交！**

## 常见问题

**Q: 构建脚本报错"字段数不匹配"怎么办？**

A: 这是格式错误。检查报错的行：
1. 查看报错信息：期望 X 列, 实际 Y 列
2. 如果是 CSV：数逗号数量，确保与表头一致
3. 如果是 MD：数 `|` 数量，确保与表头一致
4. 修复后重新运行构建

**Q: 怎么知道当前有多少列？**

A: 查看构建成功的输出：`✓ CSV 解析完成: 43 行, 36 列` —— 这里的 36 就是当前列数。

**Q: 新增厂商后怎么验证成功了？**

A: 
1. 构建成功（看到 ✅ 完成！）
2. 检查生成的 `README.md`，确认新厂商列出现
3. 检查网站是否正常显示

**Q: 为什么推荐编辑 MD 而不是 CSV？**

A: 
- MD 表格用 `|` 分隔，列边界清晰，Agent 不容易错位
- CSV 用逗号分隔，Agent 容易数错逗号数量
- MD 不需要处理引号和转义规则
- Git diff 更直观

**Q: 只想改一个链接，不动其他内容？**

A: 直接编辑 MD 文件，找到对应模型修改 URL，然后运行构建脚本。
