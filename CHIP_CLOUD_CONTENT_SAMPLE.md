## 统计数据

### 填充率

| 列 | 有内容的月数 | 填充率 |
|---|---|---|
| 模型厂商（平均） | ~25/42 | ~60% |
| LLM-Applications | ~8/42 | ~19% |
| **芯片** | **~10/42** | **~24%** |
| **云计算** | **~10/42** | **~24%** |

### 事件分布

**芯片层（共约 15 个事件）**：
- NVIDIA：4 个（H200、Blackwell、Nemotron 3、Nemotron 3 Nano/Super）
- AMD：2 个（MI300X、MI350X）
- Google：2 个（TPU v5p、TPU v6e）
- 其他：HBM3E、Trainium2、Maia 100、Ascend 910C、MI350X

**云计算层（共约 12 个事件）**：
- AWS：4 个（Bedrock、Trainium2、AI Factories、降价）

- 融资：3 个（CoreWeave 75亿/85亿、Lambda 3.2亿）
- 其他：无

## 暴露的问题

### 1. 芯片列：NVIDIA 独大

10 个有内容的月份中，NVIDIA 占 4 个。读者会误以为"只有 NVIDIA 在做事"，实际上：
- Intel Gaudi：存在感弱
- Qualcomm Cloud AI：几乎无重大发布
- 创业公司（Groq/Cerebras）：融资算芯片还是云计算？

### 2. 云计算列：融资 ≠ 服务发布

把"CoreWeave 75亿美元"和"AWS Bedrock GA"放在同一列，语义混乱：
- 一个是融资新闻
- 一个是产品发布
- 读者无法区分"发生了什么"

### 3. 多事件单元格过长

```
AMD MI300X + Google TPU v5p
```
这种格式在窄列中显示效果差，链接处理也复杂。

### 4. 中国厂商缺失

芯片列没有：
- 寒武纪（MLU）
- 海光（DCU）
- 平头哥（倚天/含光）

云计算列没有：
- 阿里云（灵骏集群）
- 华为云（昇腾云服务）
- 腾讯云（HCC 集群）

如果加中国厂商，事件会翻倍，但国际用户不关心。

## 改进建议（如果一定要做）

### 方案 A：芯片列分南北两派

```csv
Month,...,芯片-国际,芯片-中国
```

| Month | 芯片-国际 | 芯片-中国 |
|---|---|---|
| 2023-12 | AMD MI300X + Google TPU v5p | |
| 2024-12 | Google TPU v6e | Huawei Ascend 910C |

### 方案 B：云计算列分类型

```csv
Month,...,云-产品发布,云-定价/融资
```

| Month | 云-产品发布 | 云-定价/融资 |
|---|---|---|
| 2023-04 | AWS Bedrock GA | |
| 2024-05 | | CoreWeave 75亿美元 |
| 2025-06 | | AWS GPU降价45% |

### 方案 C：只记录"里程碑"（最精简）

只记录真正改变行业格局的事件：

| Month | 芯片 | 云计算 |
|---|---|---|
| 2022-11 | | Azure OpenAI |
| 2023-12 | AMD MI300X + TPU v5p | |
| 2024-03 | NVIDIA Blackwell | |
| 2024-05 | | CoreWeave 75亿 |
| 2025-06 | AMD MI350X | AWS降价45% |
| 2026-02 | | Stargate 1000亿 |

这样只有 6 个月有内容，填充率 ~14%，但信息密度高。

## 结论

**如果直接加列，实际填充的内容如下特征**：

1. **稀疏**：42 个月中约 10 个月有内容，大量空白
2. **NVIDIA 独大**：芯片列几乎变成"NVIDIA 月表"
3. **语义混杂**：融资、发布、降价混在一起
4. **中国缺失**：如果要全面，列会爆炸
5. **维护困难**：需要同时跟踪 10+ 芯片厂 + 10+ 云厂商

**我的最终建议**：

与其在现有表格中硬塞两列瘸腿，不如：
- 在 **README 底部** 加一个"基础设施大事记"段落
- 或在 **网页** 上加一个折叠面板"芯片与云"
- 记录 10-15 个真正改变格局的里程碑事件

这样既保留了信息，又不破坏表格结构。
